Change Log

Version 1.13.0

  • Minimum required DNN version is now 6.0.0
  • Fixed a bug where database connections weren't being disposed
  • Fixed an issue where extra files were included in the module package

Version 1.12.5

  • Fixed a bug where there could be an error when an application contained both a resume and cover letter

Version 1.12.4

  • Fixed a bug where some delete buttons didn't take any action
  • Fixed a bug where applicant's email address wasn't validated

Version 1.12.3

  • Fixed a bug where emails were sent from the wrong address

Version 1.12.0, Version 1.12.1, & Version 1.12.2

  • Added compatibility with the Azure environmet.

Version 1.11.1

  • Fixed a bug in 1.11.0 where resumes were never required for logged-in applicants.

Version 1.11.0

  • Minimum requirement changed to DotNetNuke 5.1.3 (from 5.1.0).
  • Include applicant's resume and cover letter in notification email.
  • Add name, email, and phone fields to application form.
  • The applicant's display name and username can be included in body of emails.
  • The resumé field can now be hidden or made optional.
  • When an applicant isn't logged in, don't try to put their name on email subject line.
  • Fixed bug where Manage Applications didn't display expand/collapse icon next to job rows.
  • Fixed bug where notification email linked to incorrect resume when an applicant updated their application.
  • Fixed bug where documents with special characters (such as a space) in the file name weren't downloaded with the correct name in some browsers.

More details can be found on the Engage: Employment Change Log
